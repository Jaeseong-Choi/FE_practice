block vs inline vs inline-block

Live Server로 열어 실제 배치 차이를 확인하세요. 각 스테이지 안의 흰색 박스가 요소들입니다.

① Block 요소

한 줄 전체를 차지하고, 항상 새 줄에서 시작합니다. width: auto일 때 부모의 가로폭을 가득 채웁니다.
display: block
.item (block) — 줄바꿈됨
.item (block) — 가로 전체 사용
.item (block) — 세로로 쌓임

특징: 자동 줄바꿈 / width 기본값 auto(부모 폭) / height, margin/padding 모두 적용

② Inline 요소

줄 안에서 흐르며 옆으로 나란히 배치됩니다. width/height를 지정해도 적용되지 않습니다.
display: inline
텍스트와 함께 .tag (inline) .tag with width/height 시도 .tag (inline) 이어집니다.

특징: 가로로 흐름 / width, height 미적용 / 좌우 margin은 적용되지만 상하 margin은 레이아웃에 영향 적음

③ Inline-block 요소

줄 안에 나란히 배치되지만, block처럼 width/height가 적용됩니다.
display: inline-block
.cardlet (inline-block)
width/height 지정 O
줄바꿈 없이 옆으로

특징: 인라인처럼 옆으로 배치 + block처럼 width/height/margin/padding 적용

사용 팁

  • block: 섹션, 카드, 문단 등 세로로 쌓이는 영역
  • inline: 텍스트 일부 강조, 링크, 아이콘 등 텍스트 흐름 안의 요소
  • inline-block: 버튼, 배지, 작은 카드처럼 옆으로 나란하지만 고정 크기가 필요한 컴포넌트

Live Server로 보는 방법

  1. 이 파일을 block-inline-inline-block-demo.html로 저장합니다.
  2. VS Code에서 Live Server 확장을 설치합니다.
  3. 편집기에서 마우스 오른쪽 클릭 → Open with Live Server.
  4. 브라우저에서 각 스테이지의 흰 박스 배치(세로로 쌓임 vs 가로 흐름 vs 가로 나란히+크기 지정)를 비교하세요.
